# MTProxy Docker Wrapper

Docker-обёртка для `TelegramMessenger/MTProxy`.

## Build

```bash
docker build -t tg-mtproxy .
```

Debug-образ с сетевыми утилитами:

```bash
docker build --build-arg DEBUG=true -t tg-mtproxy:debug .
```

## Run

Рекомендуемый запуск на VPS:

```bash
docker run -d \
  --net=host \
  --name tg-mtproxy \
  --restart=always \
  -v tg-mtproxy-data:/data \
  -e PUBLIC_HOST=YOUR_PUBLIC_IP \
  -e CLIENT_SECRET=$(openssl rand -hex 16) \
  -e TAG=YOUR_PROMO_TAG \
  -e WORKERS=4 \
  -e PORT=443 \
  -e RANDOM_PADDING=true \
  tg-mtproxy
```

Логи:

```bash
docker logs -f tg-mtproxy
```

## Env

- `PORT` default `443`
- `STATS_PORT` default `8888`
- `WORKERS` default `1`
- `CLIENT_SECRET` 32 hex chars, autogenerated if empty
- `RANDOM_PADDING` `true|false`
- `SECURE_ONLY` `true|false`
- `PUBLIC_HOST` public IP or domain for generated proxy links
- `TAG` MTProxy promo tag

## Debug

Если образ собран с `DEBUG=true`, внутри контейнера доступны:

- `iproute2`
- `iputils-ping`
- `netcat-openbsd`
- `procps`
- `telnet`
- `dnsutils`
